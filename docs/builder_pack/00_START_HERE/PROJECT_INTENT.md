# Project Intent

Heimdal Engine should make local LLM agents feel like frontier-class cloud agents through architecture, not raw model size.

The core thesis:

> A smaller local model can produce frontier-like agent outcomes if surrounded by context discipline, retrieval, tools, verification, evals, patching, hardware-aware scheduling, and continuous self-improvement loops.

Heimdal must support a single external persona/agent in a host system, internal multi-agent/subprocess orchestration, hardware portability, controlled self-improvement, optional cloud-model comparison as a teacher loop, and local-first operation.

# Contributing to Heimdal

Thank you for your interest in contributing to Heimdal! This document provides guidelines and instructions for contributing.

## How to Contribute

1. **Fork the repository** - Create a personal fork of the project
2. **Create a branch** - Create a feature branch for your changes
3. **Make changes** - Implement your feature or fix
4. **Write clear commit messages** - Explain what and why, not just what
5. **Submit a pull request** - Open a PR with a clear description

## Code Style

- Write clean, readable code
- Add comments for complex logic
- Include docstrings for functions and classes
- Follow language-specific conventions

## Documentation

- Update README.md if adding new features
- Add comments to explain your code
- Include usage examples where appropriate

## Testing

- Test your changes thoroughly
- Ensure existing functionality still works
- Add tests for new features if applicable

## Questions?

Feel free to open an issue with your question or suggestion.

Thank you for contributing!
